Understanding `:` and `::` in Rust
1
:
The Question

When learning Rust and Bevy, one of the most confusing syntax elements for beginners is the difference between : and ::.

Excellent question! This is core Rust syntax, and understanding it will help a LOT when reading Bevy code.

We'll explain : and :: slowly and precisely, using real Bevy code examples.

::
The Code We're Explaining
material: materials.add(StandardMaterial {
    base_color: Color::srgb(0.8, 0.7, 0.6),
    ..default()
}),

This code snippet contains both : and :: syntax. Let's break it down!

The Single Colon `:` (Field Assignment)
2
:
What `:` Means

In Rust, inside a struct literal, the single colon : means:

"Assign this value to this field"

It's the syntax for field assignment when creating a struct instance.

Where we see it:

  • material: materials.add(...)
  • base_color: Color::srgb(...)

Think of : as "gets" or "is assigned to" when reading code.

Code Structure
// Struct definition (simplified)
struct StandardMaterial {
    base_color: Color,
    metallic: f32,
    roughness: f32,
}
// Creating an instance with `:`
let material = StandardMaterial {
    base_color: Color::srgb(0.8, 0.7, 0.6),
    metallic: 0.0,
    roughness: 0.5,
};

Each : assigns a value to a specific field of the struct.

Example 1: `material: ...`
3
Understanding `material`

Code:

material: materials.add(...)

How to read it:

"The material field gets the value returned by materials.add(...)"

What is `material`?

  • material is a field name
  • It belongs to PbrBundle

Internally, PbrBundle has a structure like:

PbrBundle Structure
// Simplified PbrBundle definition
struct PbrBundle {
    mesh: Handle<Mesh>,
    material: Handle<StandardMaterial>,
    transform: Transform,
    // ... other fields
}
// Creating a PbrBundle instance
let bundle = PbrBundle {
    mesh: mesh_handle,
    material: material_handle, // ← This is where `:` is used
    transform: Transform::default(),
};

So material: something means: "Put something into the material field."

Example 2: `base_color: ...`
4
Understanding `base_color`

Code:

base_color: Color::srgb(0.8, 0.7, 0.6)

How to read it:

"The base_color field gets this color value"

What is `base_color`?

  • base_color is a field name
  • It belongs to StandardMaterial

StandardMaterial has a structure like:

StandardMaterial Structure
// StandardMaterial definition (simplified)
struct StandardMaterial {
    base_color: Color,
    metallic: f32,
    roughness: f32,
    // ... other fields
}
// Creating a StandardMaterial instance
let material = StandardMaterial {
    base_color: Color::srgb(0.8, 0.7, 0.6),
    metallic: 0.0,
    roughness: 0.5,
};

Again: Left of : → field name, Right of : → value

Rule for `:`
5
When to Use `:`

The single colon : has different meanings in different contexts:

1. Struct Literals (Most Common in Bevy)

Assign value to a field: field_name: value

2. Type Annotations

Specify type of a variable: let x: i32 = 5;

3. Function Parameters

Define parameter types: fn add(x: i32, y: i32) -> i32

4. Pattern Matching

In match statements: Some(x) => println!("{}", x)

In your Bevy code, when you see field_name: value inside curly braces { }, it's field assignment in a struct literal.

Summary Table
Context Example Meaning
Struct literal material: value Assign value to field
Type annotation x: i32 Variable x has type i32
Function parameters fn foo(x: i32) Parameter x has type i32
Pattern matching Some(x) => ... Destructure option

In your Bevy code, it's almost always field assignment in struct literals.

The Double Colon `::` (Path Separator)
6
::
What `::` Means

The double colon :: is a path separator in Rust.

"Access something that belongs to a type or module"

Think of it as:

  • C++ → Type::method()
  • Java/C# → Class.method() (static methods)
  • Python → module.function()

Where we see it:

  • Color::srgb(...)
  • StandardMaterial { ... } (the type name itself)
  • Transform::from_xyz(...)
Namespace Access
// Accessing associated functions
Color::srgb(0.8, 0.7, 0.6)
Transform::from_xyz(2.0, 3.0, -4.0)
Vec3::ZERO // Associated constant
// Accessing module contents
use std::fs;
fs::read_to_string("file.txt")
// Referring to a type
let material: StandardMaterial = ...;

:: connects a module/type to its contents.

Example: `Color::srgb`
7
Understanding `Color::srgb`

Code:

Color::srgb(0.8, 0.7, 0.6)

How to read it:

"Call the function srgb that belongs to the type Color"

Important Detail:

  • srgb is not a free function
  • It's an associated function of Color
  • Similar to static methods in other languages

The definition looks like:

Associated Function Definition
// Color type definition (simplified)
struct Color {
    r: f32,
    g: f32,
    b: f32,
}
// Implementation block for Color
impl Color {
    // Associated function (like a static method)
    pub fn srgb(r: f32, g: f32, b: f32) -> Color {
        Color { r, g, b }
    }
}
// Usage:
let red = Color::srgb(1.0, 0.0, 0.0);

So: Color → type, ::srgb → function attached to the type

`:` vs `::` Side-by-Side
8
:
Comparison

Let's compare : and :: directly:

Single Colon :

  • Purpose: Assign value to struct field
  • Context: Inside struct literals { }
  • Analogy: "is assigned to" or "gets"
  • Example: base_color: Color::srgb(...)

Double Colon ::

  • Purpose: Access type/module member
  • Context: Anywhere you reference a type or module
  • Analogy: "belongs to"
  • Example: Color::srgb(...)
::
Quick Reference Table
Symbol Name Purpose Example
: Field separator Assign value to struct field material: value
:: Path separator Access type/module member Color::srgb(...)

Mental Shortcut:

Single colon `:` → "is assigned to"

Double colon `::` → "belongs to"

Common Beginner Mistakes
10
Important to Avoid

Here are common mistakes beginners make with : and :::

❌ Wrong Examples:

Color:sRGB(0.8, 0.7, 0.6)

Problem: Using : instead of ::

base_color = Color::srgb(...)

Problem: Using = instead of : in struct literals

material:: materials.add(...)

Problem: Using :: instead of :

✅ Correct Examples
// ✅ CORRECT: `:` for field assignment
base_color: Color::srgb(...)
// ✅ CORRECT: `::` for type member access
Color::srgb(...)

Why these rules?

  • : assigns fields in structs (not =)
  • :: accesses type members (not :)
  • = is not used in struct literals in Rust
Field vs Type: What's the Difference?
11
Field vs Type

The Core Difference:

Fields are properties of a struct instance.

Types are categories or blueprints for creating instances.

Analogy:

  • Type = Blueprint for a house
  • Field = A specific room in a house
  • Value = What's in that room

In Rust Code:

  • StandardMaterial → A type (blueprint)
  • base_color → A field (property)
  • Color::srgb(0.8, 0.7, 0.6) → A value
Visual Representation
// TYPE: Blueprint for creating materials
struct StandardMaterial {
    // FIELDS: Properties each material has
    base_color: Color, // ← FIELD with TYPE Color
    metallic: f32, // ← FIELD with TYPE f32
    roughness: f32, // ← FIELD with TYPE f32
}
// Creating an INSTANCE of the type:
let material = StandardMaterial {
    // Assigning VALUES to FIELDS:
    base_color: Color::srgb(0.8, 0.7, 0.6), // ← VALUE
    metallic: 0.0, // ← VALUE
    roughness: 0.5, // ← VALUE
};

Key Relationship: Types define what fields exist. Fields store values. The : syntax connects fields to their values when creating instances.
